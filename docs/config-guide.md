# Just Talk - 配置文件 (config.toml) 全参数详解与模板

本指南详细介绍了 **Just Talk** 语音输入工具的配置文件参数。配置文件使用 **TOML** 格式，默认存放于各操作系统的用户配置目录下：

- **Windows**: `C:\Users\您的用户名\.config\just-talk\config.toml`
- **macOS / Linux**: `~/.config/just-talk/config.toml` (即当前用户家目录下的 `.config/just-talk/` 目录)

在程序运行过程中，如果您修改并保存了配置文件，大部分配置（如悬浮窗开启/关闭、缩放比例、大模型配置等）都会被**热加载 (Hot Reload)** 实时应用，无需重启程序。

---

## 1. 完整配置参数详解

### 1.1 `[voice]` 模块 (语音输入核心配置)

| 参数名 | 数据类型 | 默认值 | 参数作用与配置说明 | 是否全平台有效 |
| --- | --- | --- | --- | --- |
| `enabled` | Boolean | `true` | 是否启用语音输入插件。若设为 `false`，则不加载语音模块。 | 是 |
| `mode` | String | `"toggle"` | 录音触发模式。支持两类模式：<br>• `"toggle"`：点按快捷键开始，再次点按停止。<br>• `"hold"`：按住快捷键开始，松开快捷键时停止录音。 | 是 |
| `push_to_talk` | String | `"Alt+Super"` | 全局语音触发快捷键组合。<br>• 修饰键支持：`Alt` / `Ctrl` / `Shift` / `Super`（`Super` 在 Mac 下代表 `Command`/`Cmd`，在 Windows 下代表 `Win` 键）。<br>• 支持功能键：如 `"F9"`、`"Alt+F8"`。<br>• **限制**：出于全局防误触机制，系统禁止使用单一的普通字母、数字、空格或标点符号键作为录音键。 | 是 |
| `device` | String | `""` | 指定录音音频输入设备名称。留空 `""` 表示使用系统的默认录音设备。如果设备不正常，可通过 `just-talk.exe --doctor` 查阅可用麦克风设备名称列表。 | 是 |
| `gain` | Integer | `1` | 麦克风录音增益倍数（整数）。如果发现录入的声音偏小导致识别准确率偏低，可配置为 `2` 或 `3`。 | 是 |
| `stop_delay_ms` | Integer | `0` | 停止录音延时（毫秒）。例如配置为 `200` 时，在触发结束录音后，系统会多录音 200 毫秒，防止因松键过快导致句尾发生截断。 | 是 |
| `language` | String | `"zh-CN"` | 语音识别的语种。支持 `"zh-CN"`（中文）、 `"en-US"`（英文）等。 | 是 |
| `auto_submit` | Boolean | `true` | 自动上屏开关。<br>• `true`：识别出的文字会通过模拟键盘粘贴事件直接输入当前光标聚焦位置。<br>• `false`：识别出的文字仅复制到系统剪贴板，不进行模拟打字。 | 是 |
| `app_key` | String | `""` | 火山引擎（字节跳动）语音识别 (ASR) 服务的 `AppKey`（需在火山控制台获取）。 | 是 |
| `access_key` | String | `""` | 火山引擎语音识别 (ASR) 服务的 `AccessKey`（控制台个人中心密钥管理获取）。 | 是 |
| `resource_id` | String | `"volc.bigasr.sauc.duration"` | 火山引擎 ASR 大模型识别的计费资源 ID，默认时长计费。 | 是 |
| `hotwords` | Array of strings | `[]` | 语音识别专属热词库。例如 `["just-talk", "Kubernetes", "火山"]`。配置热词可以显著提高专业术语、人名和英文单词的识别准确度。 | 是 |

### 1.2 `[voice.correction]` 模块 (语音大模型纠错与优化)

| 参数名 | 数据类型 | 默认值 | 参数作用与配置说明 | 是否全平台有效 |
| --- | --- | --- | --- | --- |
| `enabled` | Boolean | `false` | 是否开启大模型润色纠错功能。开启后会先流式上屏原始草稿，结束时自动退格清除草稿，并发送请求给大模型进行语法润色，最后把优化后的完美文本一次性粘贴上屏。 | 是 |
| `api_key` | String | `""` | 火山方舟（火山引擎大模型平台）的 API Key。通常填入以 `"Bearer "` 开头的 API Key 字符串。 | 是 |
| `endpoint_id` | String | `""` | 大模型推理接入点 ID（Endpoint ID），可在火山方舟控制台创建豆包模型接入点后获取（如 `ep-xxxxxxxx-xxxx`）。 | 是 |
| `url` | String | `"https://ark.cn-beijing.volces.com/api/v3/chat/completions"` | 火山方舟 Chat Completions 接口请求地址。默认为北京节点，支持其他自定义大模型接口地址。 | 是 |
| `temperature` | Float | `0.1` | 模型生成温度（`0.0` 到 `1.0`）。建议固定设为 `0.1`，以严格限制大模型生成逻辑，仅用于纠错、润色和整理语气词，杜绝“胡言乱语”。 | 是 |
| `max_tokens` | Integer | `0` | 大模型生成最大 Token 数。设为 `0` 时，系统会自适应估算最大 Token 限制（计算方式为 `len(输入文字) * 2 + 100`），保证输入很长时也不会发生输出被截断，且节省 Token 消费。 | 是 |

### 1.3 `[overlay]` 模块 (录音状态胶囊悬浮窗)

| 参数名 | 数据类型 | 默认值 | 参数作用与配置说明 | 是否全平台有效 |
| --- | --- | --- | --- | --- |
| `enabled` | Boolean | `true` | 是否启用顶层状态指示悬浮窗。如果在 Windows 上设为 `true` 且启动，系统会弹出精致的原生毛玻璃半透明指示胶囊。 | 是 |
| `position` | String | `"bottom-center"` | 胶囊在桌面显示的位置。支持以下配置选项：<br>• `"top-left"` / `"top-center"` / `"top-right"`<br>• `"bottom-left"` / `"bottom-center"` / `"bottom-right"`<br>• `"center"`<br>在 Windows 平台下，计算底部坐标时，系统会自动排除任务栏高/宽，避开遮挡。 | 是 |
| `idle_visible` | Boolean | `false` | 在系统空闲（Idle）时，悬浮窗是否依然常驻显示在桌面。若设为 `false`，则只有在录音、连接中、优化中、出错等活跃状态下才弹出，平时完全隐藏。 | 是 |
| `scale` | Float | `1.0` | 悬浮窗大小缩放比例。例如设为 `1.5` 可以放大悬浮窗，设为 `0.8` 可以缩小悬浮窗。 | 是 |

### 1.4 `[debug]` 模块 (日志与调试)

| 参数名 | 数据类型 | 默认值 | 参数作用与配置说明 | 是否全平台有效 |
| --- | --- | --- | --- | --- |
| `enabled` | Boolean | `false` | 是否开启调试日志模式。开启后会输出详细的底层按键捕获、音频保存、ASR 解析过程等调试日志，方便定位环境问题。 | 是 |
| `hotkeys` | Array of strings | `[]` | 用于调试的快捷键列表，系统会自动输出这些热键的捕获调试状态。 | 是 |

## 2. 开箱即用的 `config.toml` 配置模板

您可以直接复制下方内容并保存到配置文件中：

```toml
# ==============================================================================
# Just Talk 全局配置文件
# 支持动态热重载：在运行时修改此文件保存，大部分参数会即时生效
# ==============================================================================

[voice]
# 启用语音模块
enabled = true

# 触发模式："toggle" (按一下开始，再按一下停止) 或 "hold" (按住录制，松开结束)
mode = "toggle"

# 全局语音唤起热键
# 修饰键支持 Ctrl, Alt, Shift, Super (Super 对应 Win 键/Cmd 键)
# 注意：不能使用普通字符、数字、空格、标点符号等做为独立热键
push_to_talk = "Alt+Super"

# 麦克风设备名。留空 "" 会自动选择系统默认录音设备。
# 如果默认设备录音不正常，可以通过命令行执行 `just-talk.exe --doctor` 查阅可用麦克风列表
device = ""

# 音频放大增益 (如果识别音量较小，可以修改为 2 或 3，默认为 1)
gain = 1

# 停止录音缓冲延迟 (毫秒)
# 设为 150-200ms 可以防止由于松开热键过快导致最后一句话的尾音录不进去
stop_delay_ms = 0

# ASR 听写语种 (zh-CN: 中文, en-US: 英文)
language = "zh-CN"

# 自动模拟打字上屏 (设为 false 时，识别出的文字仅复制到剪贴板，不模拟键盘输入)
auto_submit = true

# 火山引擎 (Volcengine) ASR 接口凭证 (火山控制台 -> 语音识别服务获取)
app_key = "您的火山引擎AppKey"
access_key = "您的火山引擎AccessKey"
resource_id = "volc.bigasr.sauc.duration"

# 语音识别专属热词库
# 针对特定英文缩写、技术术语或人名进行增强识别，防止音似字打错
hotwords = ["Windows", "PowerShell", "just-talk", "CMD", "Git"]


# ------------------------------------------------------------------------------
# 火山方舟 (豆包大模型) 语音润色与错别字优化配置
# ------------------------------------------------------------------------------
[voice.correction]
# 开启大模型优化纠错
# 录音结束时，系统将流式上屏的文本退格抹除，换用大模型精润后的完美文本一次性输入
enabled = true

# 火山方舟大模型接口 API Key (格式通常为 "Bearer 您的密钥" 或直接写密钥)
api_key = "Bearer 您的火山方舟API密钥"

# 火山方舟大模型推理接入点 ID (在火山控制台创建豆包模型接入点后获取，格式如 ep-xxxxxxxx-xxxx)
endpoint_id = "您的火山方舟推理接入点ID"

# 大模型接口地址 (北京节点，默认无需更改)
url = "https://ark.cn-beijing.volces.com/api/v3/chat/completions"

# 温度控制：温度设为较低值 (0.0 或 0.1) 可最大限度规避模型幻觉，仅作错字标点纠正和语气词整理
temperature = 0.1

# 生成最大长度：设为 0 代表根据输入文字长度动态自动计算安全大小
max_tokens = 0


# ------------------------------------------------------------------------------
# 状态指示悬浮窗 (录音状态胶囊) 配置
# ------------------------------------------------------------------------------
[overlay]
# 在桌面上开启顶层半透明磨砂质感悬浮窗 (支持 Windows/macOS/Linux)
enabled = true

# 悬浮窗显示位置
# 可选值: top-left, top-center, top-right, center, bottom-left, bottom-center, bottom-right
# 默认为 bottom-center (在 Windows 上会自动计算避开底部的系统任务栏)
position = "bottom-center"

# 系统空闲时是否常驻显示悬浮窗
# false: 仅在录音/优化/出错时显示，平时自动隐藏隐藏 (推荐)
# true: 始终常驻在屏幕顶层，空闲状态显示为灰色
idle_visible = false

# 悬浮窗大小缩放比例 (默认 1.0)
scale = 1.0


# ------------------------------------------------------------------------------
# 开发者调试与日志配置
# ------------------------------------------------------------------------------
[debug]
# 是否开启底层 verbose 级别的调试日志
enabled = false
# 捕获调试按键细节列表
hotkeys = []
```
